# Roadmap

## Objetivo

Este documento apresenta o planejamento evolutivo do SumuBank. O desenvolvimento será realizado de forma incremental, garantindo que cada fase entregue uma aplicação funcional e preparada para receber novas funcionalidades.

## Visão Geral

| Fase | Objetivo               |
|------| ---------------------- |
| 1    | Planejamento           |
| 2    | MVP                    |
| 3    | Segurança              |
| 4    | Qualidade              |
| 5    | Comunicação Assíncrona |
| 6    | Deploy                 |
| 7    | Observabilidade        |

## Fase 1 - Planejamento

Definir a arquitetura e a documentação do projeto.

Entregas

* README inicial
* Arquitetura
* Modelo de domínio
* Modelo do banco
* Requisitos funcionais
* Endpoints
* Roadmap
* Planejamento do deploy

## Fase 2 - MVP

Construir uma aplicação funcional.

Entregas

* Angular
* Spring Boot
* PostgreSQL
* Docker Compose
* Swagger
* Cadastro de clientes
* Cadastro de contas
* Depósito
* Saque
* Transferência
* Extrato

## Fase 3 - Segurança

Adicionar autenticação e autorização.

Entregas

* Spring Security
* JWT
* Login
* Cadastro
* Perfis
* Permissões

## Fase 4 - Qualidade

Garantir a qualidade da aplicação.

Entregas

* JUnit
* Mockito
* Cobertura de testes
* Refatorações
* Padronização do código

## Fase 5 - Comunicação Assíncrona

Desacoplar processos utilizando mensageria.

Entregas

* RabbitMQ
* Eventos
* Consumidores
* Notificações
* Auditoria básica

## Fase 6 - Deploy

Disponibilizar a aplicação em produção.

Entregas

* Frontend na Vercel
* Backend no Render
* PostgreSQL no Neon
* RabbitMQ (CloudAMQP ou Docker)
* AWS S3
* GitHub Actions

## Fase 7 - Observabilidade

Monitorar a aplicação em produção.

Entregas

* Spring Boot Actuator
* Micrometer
* Prometheus
* Grafana
* Health Checks
* Logs estruturados

## Futuras Melhorias

* Investimentos.
* Empréstimos.
* Agendamento de transferências.
